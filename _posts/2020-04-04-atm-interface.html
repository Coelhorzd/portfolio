Overview


The challenge

This UX design exercise was chosen after I observed people in a queue waiting to use an ATM machine.

Most used ATMs in Brazil are in the banks, but even that nowadays many people use their bank app we still have to use the ATM for some tasks. So it's a great opportunity for a project trying to improve ATM users experience.

This study case happens in Brazil so perhaps some points may only make sense for user experience at the ATM machines of brazilian banks.

My role and length of project

I have done project on my own and it took me about 5 months.

Design problems

- Withdrawing and deposit are the most frequent tasks at ATMs;

- Users would like to do as much as possible on the app, but some functions still need them to go to an ATM or even inside the bank to talk to someone, like update personal information, deal with loan details, limit adjsutments etc;

- Losing time is still a big deal for ATMs. People wait on the line, wait for the machine to recongnize their card, wait for the machine to verify their password or biometrics and they wait for the machine to select and deliver the money.



Ideation


HOW MIGHT WE...

...

shorten the waiting time?

...

simplify the access to the most used options but don't forget the less used ones (hidden under many menu levels)?



Solution

As the most frequent tasks aren't numerous we can start by bringing them to the home screen.

But as we can't let the used tasks more difficult to find we could make use of a search bar. The users can search for the ke words which will show them options related to the key word.

For the many waiting moments during the journey, the ATM could run the authentications on the background while the user may advance some steps in their tasks instead of waiting for the machine to allow them to go further.

So the result after some iterations and adjustments is:


- A simple interface with the most used services + a search bar to find the less used ones in an easier and faster way. Yet some buttons for each main service category are visible for those itens which are used from time to time;


- Regarding time efficiency the interface runs the authentications on second plan while the user would not wait to proceed the necessary steps of the desired task.
